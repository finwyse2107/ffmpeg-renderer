# ffmpeg-renderer

Tiny HTTP service that accepts a bash render script (the one generated by the n8n `Render Script` Code node) and returns the produced MP4 binary.

## Endpoint

```
POST /render
Authorization: Bearer <SHARED_SECRET>
Content-Type: application/json

{ "command": "#!/bin/bash\n..." }
```

Returns `200 video/mp4` (binary) on success, or `4xx/5xx JSON` on failure with `stderr`/`stdout` tails.

`GET /health` → `{ "status": "ok" }` (no auth).

## Deploy on Dokploy (same VPS as n8n)

1. **Push these files to a Git repo** (GitHub/GitLab) — Dokploy deploys from Git.
2. In Dokploy → **Create → Compose** (or Application from Git).
3. Set source = this repo.
4. Add environment variable `SHARED_SECRET` = a long random string. Generate one:
   ```
   openssl rand -hex 32
   ```
5. Make sure the service joins the same Docker network as n8n. The compose file uses `dokploy-network` (Dokploy's default). If your n8n stack uses a different network, change the `networks` block to match.
6. Deploy. The service will be reachable at `http://ffmpeg-renderer:3000` from inside the n8n container.

## Verify

From inside any container on the same network:
```
curl http://ffmpeg-renderer:3000/health
```
Expect `{"status":"ok"}`.

## n8n workflow integration

In the `Render Execution` node (HTTP Request):
- **Method:** POST
- **URL:** `http://ffmpeg-renderer:3000/render`
- **Authentication:** Generic Credential → HTTP Header Auth → header `Authorization: Bearer <your SHARED_SECRET>`
- **Body:** JSON
  ```json
  { "command": "={{ $json.command }}" }
  ```
- **Response → Response Format:** File
- **Response → Put Output in Field:** `data`

The `YouTube` upload node reads the binary from the `data` field automatically.

## Security notes

- The endpoint executes arbitrary shell — keep the port unpublished and rely on the Docker network for isolation.
- `SHARED_SECRET` is the only auth. Rotate if leaked.
- Default render timeout: 30 minutes (`MAX_RENDER_MS`).
- Work dirs under `/tmp/render-<id>` are removed after each render.
